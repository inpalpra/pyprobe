# REVIEW PROTOCOL

Follow this structure strictly.

## Phase 1 — Context Absorption
- Read the Product `Constitution.md`.
- Read the current architecture summary.
- Understand recent changes.
- Identify the stated intent of the latest work.

Summarize what you believe PyProbe currently is.

---

## Phase 2 — Real Usage Simulation
Mentally execute defined workflows as a serious user would.

Ask:
- Is this predictable?
- Is this explicit?
- Is this discoverable?
- Is there cognitive friction?
- Does it feel like a precision instrument?
- Where would a new serious user stumble?

Document specific friction points.

---

## Phase 3 — Alignment Check
For each major product principle:
- Is it reinforced?
- Is it weakened?
- Is it violated?
- Is it being rationalized?

Be concrete.

---

## Phase 4 — Structural Pressure Test
Evaluate:

- API symmetry and consistency
- Abstraction layering
- Hidden state creep
- Complexity growth
- DSP invariant enforcement
- Test depth (boundary + adversarial)
- Whether coverage represents depth or illusion

Call out weak rigor.

---

## Phase 5 — Ruthless Clarity Section

Produce:

### 1. What Is Strong
(Only if genuinely strong.)

### 2. What Is Fragile
(Structural risk, not cosmetic.)

### 3. What Is Misaligned
(Drift from constitution.)

### 4. What Should Be Deleted

### 5. The Single Most Important Next Move
(Not five. One.)

### 6. If You Do Nothing
